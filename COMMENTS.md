## Дорожная карта

### Фаза B: UX и визуал (P2)

- Контекстное меню (ПКМ) — Mantine `Menu`, пункты по контексту выделения
- Магнетизм оси габарита: `SnapService` + пунктирная линия в `PlacementPreview`
- `DisplaySettingsStore` — масштабы/размеры из `constants.ts` в настраиваемый стор
- Аудит `AnchorSection.getCatenaryPoses()` — edge-case с overlap-range
- Массовое создание опор: модалка-таблица (name, x, track, gabarit, side)

### Фаза C: Доменные фичи (P2)

- Inline-редактирование длины пролёта (клик → NumberInput; Shift = сдвинуть цепочку)
- Средние анкеровки — новая сущность `MediumAnchor`, ромб на линии подвески
- Редактирование параметров railway (км.пк начало/конец)

### Фаза D: Вывод (MVP)

- **Таблица-сетка и спецификация** — SVG-таблица над/под планом; заголовки = опоры, строки = габарит/тип/зигзаг

### Фаза E: Рефакторинг (P3–P4)

- Класс `Anchor { type, direction }` вместо `anchorGuy`/`anchorBrace`
- Профилирование рендеринга (per-entity observer-подкомпоненты, React DevTools)
- XLSX импорт опор (SheetJS)
- Трансформаторы/разъединители (новые типы сущностей + Layer)
- Rename `PolesStore` → `CatenaryPolesStore`

## Открытые вопросы моделирования

- **Сдвоенные опоры + ригель**: рекомендуется `stanchionCount: 1|2` в `CatenaryPole` (покрывает 95% случаев, визуал — 2 круга рядом). Альтернатива: `PoleAssembly { stanchions[] }` — гибче, но сложнее.
- **Анкер как отдельная сущность**: пока свойство опоры. Сценарий "демонтированный анкер" редок — при необходимости `OrphanAnchor`.
